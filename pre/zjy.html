01 · Build independent evidence

Three Retrieval Scores

01

Sentence embedding similarity

Encodes the whole user question and compares it with pre-computed FAQ question embeddings. Unit-normalized embeddings make the dot product behave like cosine similarity.

sembed(i) = q · di= cos(q, di)q: user question · di: FAQ question i
02

TF-IDF sentence similarity

Transforms the question into a TF-IDF vector and compares it with all FAQ questions. This captures exact words, short phrases, and lexical anchors.

stfidf(i) = vq · vi||vq|| ||vi||Cosine similarity over TF-IDF vectors
03

Token-level embedding similarity

Removes stopwords, compares every query token with candidate question + keyword tokens, then averages each token's best semantic match.

stoken(i) = 1m Σa=1m best(ta, Ti)best(ta, Ti) = maxb cos(e(ta), e(cb))Average each query token's best candidate-token match

02 · Fuse by rank

Three-way Score Fusion

retrieval.py uses Reciprocal Rank Fusion, so each method votes by ranking instead of contributing raw scores on incompatible scales.

If one FAQ stays near the top in semantic, lexical, and token rankings, it receives a high final score.

04 · Refuse weak evidence

Fallback Logic

The retriever does not force an FAQ answer when the evidence is too thin.

“Sorry, I do not have an answer for that.”
01

No lexical anchor from important keyword or TF-IDF overlap

06 · Case study

The “English classes” error

A short query can contain a broad lexical anchor without enough context to separate similar FAQ intents.

Query: English classes
0.968score
Expected · AA_015

What should I do if I can't understand the English-only instruction at the beginning? Any adaptation tips?

Retrieved · AA_017

What are the English language requirements for international students applying to foundation programmes?

The short query only contains a broad lexical anchor, “English”. The retriever confuses English-medium class adaptation with English language requirements because both candidates share semantically similar terms but refer to different student needs.
